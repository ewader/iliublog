从WordPress转到Hugo

在cloudcone上买的VPS马上就要到期了，由于去年买的是黑五预售，比正经黑五的机子要贵20来块钱，而且配置还要低一些。所以，续费是不能续费的了。原本打算在荷兰的liteserver这个上面暂时放一段时间。这是一台2c2g的机子，装完php、MySQL、和Nginx之后，只要运行一段时间，内存就基本上爆了。再加上最近lnmp和oneinstack都被收购而且被挂恶意代码，也不想用宝塔之类的面板，自己编译环境又很麻烦。于是，就产生了转成静态博客的想法。在这里也要感谢阿甘和韩宇的帮助。 原本以为很复杂，其实操作下来发现也是挺简单的。 首先，WordPress的文章转换成md文件 这里有大佬已经整好了，我是使用的插件的形式。 在WordPress的插件目录wp-content/plugins/下，直接克隆WordPress to Hugo Exporter的源码。 $ git clone https://github.com/SchumacherFM/wordpress-to-hugo-exporter.git 然后进到网站的后台，在插件里启用这个插件。在工具里，点Export to Hugo。会下载一个压缩包，这个压缩包就是你网站的文章转换成的md文件。 这个插件很好用，能保留网站原来的目录，换成Hugo后，搜索引擎的收录不会受到影响。 第二步，安装Hugo 我是用的系统是Ubuntu22.04，所以直接安装就好了。 sudo snap install hugo hugo new site youweb 然后进入到你的网站文件夹，找一个喜欢的主题。一般主题都会有使用的方法，按照使用方法安装即可。 另外主题里面也都会有网站设置的样板，把样板文件hugo.yml放到网站的根目录，按照自己网站的域名啥的设置一下。 安装设置完成后，把你下载的文件，md文件copy到content这个文件夹，图片之类的放到static。然后 hugo server 打开浏览器，localhost：1313，文章是不是显示出来了？ 第三，生成HTML文件，并上传 使用代码 hugo 然后把public这个文件夹的内容传到你的虚拟主机、VPS或者GitHub，Cloudflare page等，再绑定域名就可以了。 至于如何操作，网上的教程大把的，直接参考就行了。 测试一下代码高亮 代码： <link href="/artalk/Artalk.css" rel="stylesheet"> <script src="/artalk/Artalk.js"></script> <!-- Artalk --> <div id="Comments"></div> <script> Artalk.init({ el: '#Comments', pageKey: '{{ .Permalink }}', pageTitle: '{{ .Title }}', server: '{{ $.Site.Params.artalk.server }}', site: '{{ $.Site.Params.artalk.site }}', // ....

回归简单之笔记软件

早些的时候，我对笔记软件是没有什么需求的。后来折腾了群晖后，就适用群晖的dsnote记录一些东西，感觉还不错。记得是2022年的5月，在酒店隔离，适逢博客群里大谈笔记软件之道，什么第二大脑，反链笔记，卡片笔记，弄的人心痒痒的，从那时候起，就开始折腾笔记软件的不归路。 trilium 最初是从这个笔记软件折腾的，记得当时群里阿均、陈仓颉、老张们一致推荐。数据库存储数据，查询方便，支持反链…优点很多，缺点也不少，最大的缺点是学习曲线陡峭、没有移动客户端。这个软件一直用到了2023年的6月份。由于实际的需求，需要在移动端记录或者查看一下笔记，trilium就有点力不从心了。于是，又开始了折腾之旅。最终在joplin和思源笔记中选择了思源笔记。至于选择的原因，可以看我之前的一篇文章笔记软件从trilium到思源or Joplin？。 思源笔记 思源笔记说实话还是可以的，基本功能是够了，开发者很勤奋，一周至少发布一个小版本，有时候甚至是两个。但是，事情往往过犹不及，版本的更新是一边修复bug一边又在制造bug。最重要是，开发者格局不大，总让人感觉一股小家子气。主要是表现对ducker版本的态度上。从6月份到十月份，短短4个月的时间，docker版本的部署方式代码至少改变了两次。关于这点，可以看我之前发的博客：思源笔记群晖正确的部署方法，这个是我7月份发的。最近新的版本，这个方法又不行了，又要加一些额外的命令行。论坛里还出现过要求docker收费的帖子，理由是docker用户只白嫖不做贡献。虽然被开发者否决了，但是无风不起浪。从开发者过一段时间就搞搞docker的部署方式来看，他就不想让你用docker版本，所以给你增加无谓的操作难度。 anytype 在适用思源的这几个月里，经过陈仓颉的介绍，还适用了anytype这个笔记软件，这个软件更复杂，可以说颠覆了笔记的传统方式，局域网里各端能秒同步。界面也很漂亮，但是，现在还是不能作为主力的笔记来使用，特别是文章想插入外链，用这个软件就更不合适了，因为它的理念是万物都是type，所以外链也是type，于是它就把你的外链转成内链，给你专门生成一条笔记。所以，我们写文章，插入外链图片，引用外链地址，在粘贴到博客的时候就很不方便。暂时就观望吧，看能不能完善。 joplin 转来转去，还是转回了joplin。这是经过我的思考后做出的决定： 首先，我不需要什么第二大脑之类的软件，来辅助我记忆，事实上它能帮我记忆的很有限。 给予上面那一点，我对双链的需求就没有那么强烈，尽管joplin也能实现双链 我需要的是一款： 同步方便的笔记软件，各个端完善 收集材料要方便，支持网页剪藏 我需要的时候能方便的搜索出来我要的内容 需要稳定，不想再折腾 综上之后，我发现，joplin是最合适的了，从6月份试用到现在，群晖里的docker套件从来没有挂过，各个客户端齐全，开源，同步方便。 目前的方案是：群晖里用docker部署joplin的同步服务服务器，使用Tailscale来进行各端的同步。当然这个方案还不够简单，比如Joplin可以直接使用onedrive进行同步，但是我认为，我目前的这个方案是综合数据安全和简单的最优化方案了，不接受反驳。

干眼病（DED）是一种令人痛苦的眼部疾病。由于其多因素的性质，DED的临床和生物学标志物可能不一致，有时与症状不符。因此，不存在确定DED严重程度的金标准模型，这可能影响治疗决策并使疾病进展的评估复杂化，特别是在临床试验的严格背景下。 ODISSEY欧洲共识小组是一个由每天处理眼表疾病问题的眼科医生组成的国际小组。该小组的目的是建立一个明确而实用的算法，用于评估和诊断严重的DED。通过采用基于共识的方法，他们评估了14个常用的DED严重程度标准。小组一致认为，在确认DED诊断后，只有两个标准，基于症状的评估和角膜荧光素染色，就足以在大多数患者中诊断出严重DED的存在。在体征和症状不一致的情况下，建议使用其他决定因素进行进一步评估。
